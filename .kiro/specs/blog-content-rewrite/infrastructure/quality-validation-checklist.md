## 1. Content Quality Standards

### Uniqueness & Originality
- [ ] **No template language remains** - Verified no forbidden phrases present
  - [ ] No "In the rapidly evolving landscape"
  - [ ] No "stands out as a pivotal subject"
  - [ ] No "maximizing asset value" (as generic goal)
  - [ ] No "The true measure of success"
  - [ ] No "multi-faceted approach"
  - [ ] No "As we look towards the future"
  - [ ] No generic section headings (The Current Landscape, Key Challenges, etc.)

- [ ] **Content is topic-specific** - All paragraphs relate directly to article subject
- [ ] **No duplicate content** - Compared to other articles, no shared paragraphs (>50 words)
- [ ] **Unique examples** - Includes specific examples not used in other articles

### Introduction (150-250 words)
- [ ] **Compelling hook** - Opens with topic-specific pain point, statistic, or question
- [ ] **Clear context** - Explains why topic matters now
- [ ] **Preview provided** - Tells reader what they'll learn
- [ ] **No generic opening** - Avoids template language
- [ ] **Word count met** - 150-250 words

### Main Body Content (800-1000 words)
- [ ] **Sufficient depth** - Covers topic comprehensively
- [ ] **Specific examples included** - At least 2-3 concrete examples or case studies
- [ ] **Data and statistics** - Includes relevant numbers, percentages, or metrics
- [ ] **Actionable recommendations** - Provides implementable advice
- [ ] **Industry terminology** - Uses appropriate technical terms for category
- [ ] **Logical flow** - Sections connect naturally
- [ ] **Paragraph length** - Most paragraphs are 3-5 sentences
- [ ] **No vague statements** - All claims are specific or supported

### Conclusion (100-150 words)
- [ ] **Specific takeaways** - Summarizes 3-5 key points from article
- [ ] **Forward-looking** - Includes brief future perspective
- [ ] **Appropriate CTA** - Call-to-action is varied and relevant
- [ ] **No repetition** - Doesn't just repeat introduction
- [ ] **Word count met** - 100-150 words

### Overall Content
- [ ] **Minimum word count** - At least 1200 words (excluding frontmatter)
- [ ] **Target word count** - Ideally 1500-2000 words
- [ ] **Professional tone** - Appropriate for C-level executives
- [ ] **Active voice** - Predominantly uses active voice
- [ ] **Confident language** - Avoids hedging (might, could, possibly)
- [ ] **No excessive self-promotion** - Balanced brand mentions

## 2. Structure & Formatting

### Heading Structure
- [ ] **H1 present** - Title appears as H1 in content
- [ ] **4-6 H2 sections** - Main sections use H2 tags
- [ ] **6-12 H3 subsections** - Subsections use H3 tags
- [ ] **Topic-specific headings** - All headings relate to article topic
- [ ] **No generic headings** - Avoids "Strategic Approaches", "Looking Ahead", etc.
- [ ] **Keyword in headings** - Primary keyword in at least one H2
- [ ] **Heading hierarchy** - Proper nesting (H2 → H3 → H4)
- [ ] **Heading length** - H2s under 60 characters when possible

### Lists & Formatting
- [ ] **Lists used appropriately** - For 3+ related items
- [ ] **Parallel structure** - List items follow consistent format
- [ ] **Bullet vs. numbered** - Correct list type for content
- [ ] **Bold/italic used sparingly** - Emphasis only where needed
- [ ] **No excessive formatting** - Clean, readable presentation

### Content Organization
- [ ] **Clear sections** - Each section has distinct focus
- [ ] **Smooth transitions** - Sections connect logically
- [ ] **Balanced length** - Sections are roughly similar in length
- [ ] **Subsections support main** - H3s relate to parent H2

## 3. SEO Optimization

### Excerpt (Frontmatter)
- [ ] **Unique excerpt** - Different from all other articles
- [ ] **Appropriate length** - 150-200 characters
- [ ] **Includes primary keyword** - Keyword present naturally
- [ ] **Compelling** - Encourages clicks
- [ ] **Complete thought** - Not cut off mid-sentence
- [ ] **No template language** - Not generic description

### Keyword Integration
- [ ] **Primary keyword identified** - Clear from title
- [ ] **Keyword in introduction** - Appears in first 100 words
- [ ] **Keyword in H2** - Present in at least one main heading
- [ ] **Keyword in conclusion** - Appears in final section
- [ ] **Appropriate density** - 1-2% of total word count
- [ ] **Natural integration** - Keywords flow naturally
- [ ] **Secondary keywords** - 3-5 related terms included
- [ ] **Long-tail variations** - Includes keyword variations

### Internal Linking
- [ ] **2-4 internal links** - Links to related blog posts
- [ ] **Descriptive anchor text** - Not "click here" or generic
- [ ] **Contextually relevant** - Links add value to reader
- [ ] **Natural placement** - Links within main content flow
- [ ] **Working links** - All links are valid (if verifiable)

### Featured Snippet Optimization
- [ ] **Snippet opportunity identified** - At least one section optimized
- [ ] **Concise answer provided** - 40-60 words for definitions
- [ ] **Question format used** - Some H2s are questions
- [ ] **List format used** - Numbered/bulleted lists for steps
- [ ] **Clear structure** - Easy for search engines to parse

### Meta Elements
- [ ] **Title optimized** - Includes primary keyword
- [ ] **Title length** - Under 60 characters
- [ ] **Excerpt optimized** - Functions as meta description
- [ ] **Category appropriate** - Matches content focus

## 5. Technical Requirements

### Markdown Syntax
- [ ] **Valid markdown** - No syntax errors
- [ ] **Proper heading syntax** - Uses #, ##, ### correctly
- [ ] **List syntax correct** - Proper bullet/numbered format
- [ ] **Link syntax valid** - [text](url) format correct
- [ ] **No HTML tags** - Uses markdown only (unless necessary)
- [ ] **Code blocks** - Uses ``` for code (if applicable)

### Frontmatter
- [ ] **All fields present** - title, date, excerpt, category, author, coverImage
- [ ] **YAML syntax valid** - Proper formatting
- [ ] **Title matches** - Frontmatter title matches H1
- [ ] **Date preserved** - Original publication date maintained
- [ ] **Date format correct** - YYYY-MM-DD format
- [ ] **Category valid** - Matches one of defined categories
- [ ] **Author correct** - "A&A Hospitality"
- [ ] **Cover image path** - Valid path format
- [ ] **Quotes escaped** - Special characters handled properly

### File Management
- [ ] **Filename unchanged** - Original kebab-case name preserved
- [ ] **UTF-8 encoding** - File saved with UTF-8
- [ ] **Unix line endings** - LF not CRLF
- [ ] **No BOM** - No byte order mark
- [ ] **File location** - In src/content/posts/ directory

## Common Issues & Solutions

### Issue: Word count too low
**Solution**: Expand sections with more examples, add subsections, include more actionable recommendations

### Issue: Content too generic
**Solution**: Add specific statistics, name technology platforms, include case studies, use concrete examples

### Issue: Template language found
**Solution**: Rewrite affected sections completely, use topic-specific language, add specific details

### Issue: Headings too generic
**Solution**: Incorporate topic keywords, make headings more specific, use question format

### Issue: Poor keyword integration
**Solution**: Rewrite sentences to include keywords naturally, use variations, check density

### Issue: No internal links
**Solution**: Identify related articles, add contextual links with descriptive anchor text

### Issue: Excerpt too generic
**Solution**: Rewrite to include specific value proposition, add primary keyword, make compelling
