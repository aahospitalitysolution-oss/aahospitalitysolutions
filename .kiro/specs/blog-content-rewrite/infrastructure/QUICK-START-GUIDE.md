# Quick Start Guide: Blog Content Rewrite

## Purpose
This guide provides a streamlined workflow for rewriting each of the 25 blog articles. Follow these steps to ensure consistent quality and efficiency.

---

## Article Rewrite Workflow

### Phase 1: Preparation (30-45 minutes)

#### Step 1: Select Article (2 minutes)
- Choose next article from prioritized list in tasks.md
- Open the markdown file
- Note the title, category, and current structure

#### Step 2: Keyword Research (15-20 minutes)
Follow `keyword-research-process.md`:

1. **Identify Primary Keyword** (5 min):
   - Extract from article title
   - Validate with Google search
   - Document in research notes

2. **Find Secondary Keywords** (5-7 min):
   - Use Google autocomplete
   - Check "People also ask"
   - Note "Related searches"
   - Select 3-5 relevant terms

3. **Discover Long-Tail Keywords** (3-5 min):
   - Question-based phrases
   - Specific scenarios
   - Location-based (if relevant)
   - Target 3-5 variations

4. **Quick Competitive Analysis** (2-3 min):
   - Google primary keyword
   - Review top 5 results
   - Note content gaps
   - Identify differentiation opportunity

**Output**: Keyword research template filled out

#### Step 3: Topic Research (30-45 minutes)

**Industry Data** (15-20 min):
- [ ] Find recent statistics (within 2 years)
- [ ] Identify current trends
- [ ] Note industry benchmarks
- [ ] Gather specific examples

**Technology/Tools** (10-15 min, if applicable):
- [ ] Research relevant platforms
- [ ] Note vendor names and features
- [ ] Find pricing information
- [ ] Identify case studies

**Regional Context** (5-10 min, if relevant):
- [ ] Southeast Asia-specific data
- [ ] Regional trends
- [ ] Local market considerations

**Sources to Check**:
- Industry publications (Hotel Management, Hospitality Net, Skift)
- Research firms (STR, Phocuswright, Deloitte)
- Technology vendors (case studies, whitepapers)
- Competitor blogs (for gaps and opportunities)

**Output**: Research notes with specific data, examples, and sources

---

### Phase 2: Writing (60-90 minutes)

#### Step 4: Create Article Outline (10 minutes)

Using the appropriate template:

1. **Introduction**:
   - Hook idea
   - Context points
   - Preview statement

2. **Main Sections** (4-6 H2s):
   - Section 1: [Topic]
   - Section 2: [Topic]
   - Section 3: [Topic]
   - Section 4: [Topic]
   - Section 5: [Topic]
   - Section 6: [Topic]

3. **Subsections** (2-3 H3s per H2):
   - List key subsection topics

4. **Conclusion**:
   - Key takeaways (3-5)
   - Forward-looking statement
   - CTA

**Output**: Structured outline with all headings

#### Step 5: Write Introduction (15 minutes)

**Requirements**:
- 150-250 words
- Compelling hook (statistic, question, or challenge)
- Clear context (why this matters now)
- Preview of what reader will learn
- Include primary keyword in first 100 words

**Template**:
```markdown
[Hook: 1-2 sentences with specific data or question]

[Context: 2-3 sentences on current landscape and relevance]

[Preview: 1-2 sentences on what article covers and benefits]
```

**Check**:
- [ ] No template language
- [ ] Topic-specific content
- [ ] Primary keyword included
- [ ] Engaging and relevant

#### Step 6: Write Main Body (40-60 minutes)

**For Each Section**:

1. **Write H2 Heading**:
   - Topic-specific, not generic
   - Include keywords where natural
   - Under 60 characters

2. **Write Section Content**:
   - 2-3 paragraphs introducing section
   - Specific examples or data
   - Actionable recommendations

3. **Write H3 Subsections**:
   - 2-3 subsections per H2
   - Detailed explanations
   - Bullet points for lists (3+ items)
   - Specific metrics or benchmarks

**Content Elements to Include**:
- [ ] Specific statistics or data points
- [ ] Industry examples or case studies
- [ ] Technology platforms (if applicable)
- [ ] Implementation steps
- [ ] Common pitfalls to avoid
- [ ] Best practices

**Writing Tips**:
- Use active voice
- Keep paragraphs 3-5 sentences
- Include specific numbers and percentages
- Name specific vendors/platforms
- Provide actionable advice
- Avoid template language

#### Step 7: Write Conclusion (10 minutes)

**Requirements**:
- 100-150 words
- 3-5 specific key takeaways (bullet points)
- Forward-looking statement (1-2 sentences)
- Varied call-to-action (1-2 sentences)
- Include primary keyword

**Template**:
```markdown
## [Conclusion Heading]

[Topic-specific summary of main strategies/insights]:

- **Takeaway 1**: [Specific insight with metric]
- **Takeaway 2**: [Specific insight with metric]
- **Takeaway 3**: [Specific insight with metric]
- **Takeaway 4**: [Specific insight with metric]
- **Takeaway 5**: [Specific insight with metric]

[Forward-looking statement about future trends or developments]

[Call-to-action that's relevant to topic]
```

**Check**:
- [ ] Summarizes specific insights
- [ ] No repetition of introduction
- [ ] Includes forward-looking perspective
- [ ] Appropriate CTA

### Phase 3: Optimization (20-30 minutes)

#### Step 8: Create Unique Excerpt (5 minutes)

**Requirements**:
- 150-200 characters
- Includes primary keyword
- Compelling and specific
- Complete thought
- Different from all other excerpts

**Process**:
1. Summarize article's main value proposition
2. Include primary keyword naturally
3. Make it compelling (encourage clicks)
4. Check character count
5. Verify uniqueness

**Example**:
```yaml
excerpt: "Discover proven strategies to increase RevPAR beyond basic rate adjustments, including ancillary revenue optimization and demand forecasting techniques."
```

#### Step 9: Add Internal Links (5 minutes)

**Requirements**:
- 2-4 links to related blog posts
- Descriptive anchor text (not "click here")
- Contextually relevant
- Natural placement in content flow

**Process**:
1. Identify related articles in blog
2. Find natural link opportunities in content
3. Use descriptive anchor text
4. Add markdown links

**Example**:
```markdown
For more on optimizing your distribution strategy, see our guide to [direct booking strategies](/blog/direct-booking-strategies-reducing-ota-dependency).
```

#### Step 10: Optimize for Featured Snippets (5 minutes)

**Opportunities**:
- Definition sections (40-60 words)
- Question-format H2s with concise answers
- Numbered lists (how-to steps)
- Comparison tables

**Process**:
1. Identify 1-2 snippet opportunities
2. Format content appropriately
3. Ensure clear, concise answers

**Example**:
```markdown
## What is RevPAR and How is it Calculated?

RevPAR (Revenue Per Available Room) is a key performance metric in hospitality that measures a hotel's ability to fill rooms at an average rate. It's calculated by dividing total room revenue by the number of available rooms, or by multiplying average daily rate (ADR) by occupancy percentage.
```

#### Step 11: Verify Keyword Integration (5 minutes)

**Check**:
- [ ] Primary keyword in introduction (first 100 words)
- [ ] Primary keyword in at least one H2 heading
- [ ] Primary keyword in conclusion
- [ ] Keyword density 1-2% (use word counter)
- [ ] Secondary keywords distributed naturally
- [ ] Long-tail keywords in subsections
- [ ] Keywords flow naturally (not forced)

**Tool**: Use word counter to check density

### Phase 4: Quality Review (15-20 minutes)

#### Step 12: Run Quality Checklist (15 minutes)

Use `quality-validation-checklist.md`:

**Content Quality**:
- [ ] No template language remains
- [ ] All headings are topic-specific
- [ ] Includes specific examples/data
- [ ] Provides actionable recommendations
- [ ] Professional tone throughout
- [ ] Word count exceeds 1200 words

**Structure**:
- [ ] 4-6 H2 sections
- [ ] 6-12 H3 subsections
- [ ] Logical flow between sections
- [ ] Paragraphs are 3-5 sentences

**SEO**:
- [ ] Unique excerpt (150-200 chars)
- [ ] Primary keyword properly integrated
- [ ] 2-4 internal links included
- [ ] Featured snippet opportunity present

**Technical**:
- [ ] Markdown syntax valid
- [ ] Frontmatter complete
- [ ] Title matches filename
- [ ] Date preserved
- [ ] Category correct

**Category Alignment**:
- [ ] Content matches category focus
- [ ] Uses appropriate terminology
- [ ] Includes category-specific elements

#### Step 13: Proofread (5 minutes)

**Check**:
- [ ] Spelling errors
- [ ] Grammar issues
- [ ] Punctuation
- [ ] Consistent terminology
- [ ] Readability

**Tools**: Grammarly, Hemingway Editor, or similar

---

### Phase 5: Finalization (5 minutes)

#### Step 14: Final Validation (3 minutes)

**Quick Checks**:
- [ ] File saved with UTF-8 encoding
- [ ] Filename unchanged
- [ ] All frontmatter fields present
- [ ] No HTML tags (markdown only)
- [ ] Links are valid format

#### Step 15: Document Completion (2 minutes)

**Record**:
- Article title and filename
- Date completed
- Word count
- Any notes or issues
- Time spent

## Common Mistakes to Avoid

### Content Mistakes:
❌ Using template language or generic phrases
❌ Vague statements without supporting data
❌ Generic headings like "Strategic Approaches"
❌ No specific examples or case studies
❌ Forgetting to include actionable recommendations

### SEO Mistakes:
❌ Excerpt is generic or too similar to others
❌ Keywords forced unnaturally into content
❌ No internal links included
❌ Primary keyword missing from key locations
❌ Keyword density too high (stuffing)

### Technical Mistakes:
❌ Changing filename
❌ Modifying publication date
❌ Incomplete frontmatter
❌ Invalid markdown syntax
❌ Using HTML instead of markdown

## Tips for Efficiency

### Speed Up Research:
- Bookmark key industry sources
- Create a vendor/platform reference list
- Save common statistics and benchmarks
- Build a library of case studies

### Speed Up Writing:
- Use templates consistently
- Write in focused blocks (Pomodoro technique)
- Don't edit while writing (write first, edit later)
- Reuse structure patterns across similar articles

### Speed Up Optimization:
- Create excerpt after writing (easier with full context)
- Add internal links as you write (when you think of them)
- Use find/replace to check keyword placement
- Keep quality checklist open while writing

## Troubleshooting

### Problem: Can't find specific data or statistics
**Solution**: Use industry estimates, cite sources when available, focus on providing best available information

### Problem: Article topic is too broad
**Solution**: Narrow focus to specific aspect, use subsections to organize, prioritize most valuable information

### Problem: Running over time budget
**Solution**: Focus on core content first, optimize later, use template more strictly, improve research efficiency

### Problem: Word count too low
**Solution**: Add more examples, expand subsections, include additional strategies, provide more implementation detail

### Problem: Content feels generic
**Solution**: Add specific statistics, name technology platforms, include case studies, use concrete examples
